## Регистрация запросаАлгоритм регистрации запроса:-В верхнем меню нажмите кнопку «Создать» и выберите тип «Запрос».![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов.png)-В форме добавления запроса заполните необходимые поля.![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_1.png)Поля Формы добавления:-Заявитель — выберите контрагента (ФИО сотрудника, который обратился по поводу сбоя, либо отдел). При выборе можно воспользоваться поиском: начните вводить ФИО или название, и затем выберите нужный вариант из предложенных системой совпадений-**Контактное лицо** — атрибут отображается в форме добавления, если в поле «Заявитель» выбрана компания; данное поле нужно, чтобы выбрать пользователя, с которым будет происходить взаимодействие по запросу, если предоставление услуги по запросу касается компании-поставщика или заказчика-Соглашение/Услуга — услуга, при использовании которой возникли ошибки/сбои (услуга, связанная с соглашением, получателем которого является автор запроса)-Тип объекта — выберите тип объекта (инцидент/запрос на изменение)-Тема — краткое описание возникшего сбоя или ошибки-Описание — подробное описание возникшего сбоя или ошибки. В описание можно вставлять скриншоты из буфера обмена (Ctrl+V)-Оборудование — выберите оборудование в пользовании контрагента (сотрудника) или во владении контрагента (отдела), с которыми связан запрос. Расширенная форма добавления связей открывается при нажатии на иконку ![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_2.png) в поле параметра-Приложения — выберите программные ресурсы в пользовании контрагента (сотрудника) или во владении контрагента (отдела), с которыми связан запрос. Расширенная форма добавления связей открывается при нажатии на иконку ![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_2.png) в поле параметра-Срочность — выберите срочность решения этого вопроса. Срочность влияет на приоритет запроса и регламентное время её решения-**Расположение** — выберите месторасположение объекта заявки-**Помещение** — выберите помещение, в котором установлен объект заявки-Прикрепить файл — при необходимости к запросу можно прикрепить файлы с ПК-Информация о заявителе — отображает контактные данные из профиля заявителя, если в поле «Заявитель» выбран сотрудник или контактное лицо-**Рекомендуемые статьи** — отображает статьи базы знаний, связанные с выбранной услугой; если с услугой не связано ни одной статьи БЗ, то поле будет пустым-Подписчики — добавьте сотрудников, которым важно получать уведомления о ходе решения запросаПоля, отмеченные красной звёздочкой, обязательны для заполнения. Форма добавления у разных типов заявок может отличаться и иметь свой перечень параметров для заполнения.

В открывшейся форме можно изменить:-Заявителя-Соглашение/Услугу-Тип запроса-Расположение-Помещение-Срочность-Также в форме отобразятся динамические атрибуты в рамках выбранной Услуги, если они в неё были добавлены (подробнее см. [«Динамические поля»).](Dizayn uslug.htm#_Динамические_поля)![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_9.png)
## Приём запроса в работуЧтобы самостоятельно приступить к обработке запроса, откройте карточку запроса и нажмите кнопку «В работу». После этого действия пользователь, взявший заявку в работу, будет автоматически назначен ответственным за эту заявку.![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_10.png)
### Передача запроса другому ответственномуЧтобы передать запрос другому ответственному, в карточке запроса нажмите кнопку «Изменить ответственного». В открывшейся форме изменения ответственного выберите сотрудника или команду, укажите причину смены ответственного в поле «Текст» и нажмите кнопку «Сохранить».Ответственным за запрос будет назначен выбранный сотрудник/команда, статус запроса не изменится, к запросу добавится комментарий с текстом из формы.![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_11.png)Поля формы изменения ответственного:-Ответственный — укажите нового ответственного (взять себе или передать своей команде можно быстрыми кнопками «себе» и «своей команде»)-Текст комментария — введите указание причины смены ответственного, либо всю необходимую информацию. В текст можно вставить упоминание:Другого сотрудника — после ввода символа «@» выберите сотрудника. Всем выбранным сотрудникам будет отправлено оповещение с текстом комментария. По имени сотрудника из комментария можно будет перейти в карточку сотрудника- Статью Базы знаний — после ввода строки «БЗ» выберите статью Базы знаний. По названию статьи из комментария можно будет перейти в карточку статьи-Приватный — если флажок стоит, то содержание комментария видно только лицензированным сотрудникам-Дополнительные файлы — при необходимости прикрепите дополнительные файлы-Уведомить внешний email — уведомление получателей вне компании (в этом поле надо указать почту получателей, если их несколько, то перечислить адреса через запятую)
### Подписка на запросЛицензированные сотрудники могут подписаться на запрос. Подписчики запроса получают оповещения о событиях, связанных с запросом (добавление комментария, смена статуса). Также, подписчики могут отписаться от запроса.Чтобы подписаться на запрос, в карточке запроса нажмите кнопку «Подписаться», чтобы отписаться — кнопку «Отписаться».![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_12.png)
## Согласование запросаПри выполнении работ по запросу на обслуживания может потребоваться её согласование с заинтересованными лицами. Чтобы согласовать запрос, нажмите кнопку «Согласовать» и заполните необходимые поля. Запрос получит статус «На согласовании». Если нужно добавить ещё одно согласование, перейдите в раздел «Согласования» и нажмите «Добавить». Когда все согласования завершены и получены резолюции, статус запроса автоматически меняется на «В работе».![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_13.png)В открывшейся форме выберите согласующих сотрудников и укажите период согласований. После сохранения согласование отобразится в карточке согласующего сотрудника. Подробнее см. раздел [«Работа с согласованиями».](Rabota s soglasovaniyami.htm#_Работа_с_согласованиями)
## Выполнение запросаПосле завершения работ по запросу ответственный должен отчитаться о выполнении запроса и перевести её в статус «Выполнен».Для этого в карточке запроса нажмите кнопку «Выполнить», заполните поля формы изменения статуса и нажмите кнопку «Сохранить».![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_14.png)Поля формы:-Код решения — выберите код решения из списка справочника. Например: «Выполнено «Дубль», «Отклонить», «Просрочено», «Согласовать», «Эскалировать» и прочее-Отчёт об устранении инцидента/выполнении запроса — введите содержание отчёта специалиста о завершении работ по запросу-Затраченное время (в часах) — введите затраченное время. Оно автоматически отобразится в трудозатратах, связанных с этим запросом-**Добавить статью БЗ на основе отчёта о выполнении**:Флажок установлен — в Базу знаний добавляется статья, содержащая отчёт о работе по запросу- Флажок снят — статья не добавляется-В раздел — укажите раздел Базы знаний, в который необходимо добавить статью-Название статьи — введите название статьи
## Работа с массовыми запросамиСведения о массовости запроса на обслуживание отображаются иконками в общем списке запросов и в карточке запроса в блоке «Состояние»:|![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_7.png)|Массовый запрос, ведущая заявка|||![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_8.png)|Массовый запрос, подчинённая заявка||В карточке подчинённого запроса во вкладке «Массовые инциденты» отображается ведущий запрос.![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_15.png)Также ведущий запрос можно посмотреть в блоке «Состояние».![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_16.png)В карточке ведущего запроса во вкладке «Массовые инциденты» отображаются все массовые запросы со схожей проблемой.![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_17.png)Чтобы посмотреть все подчинённые запросы, в карточке ведущего запроса откройте вкладку «Подчинённые инциденты».![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_18.png)
### Массовые запросыЛюбому запросу можно присвоить признак «Массовый запрос».Массовый запрос — это запросы, вызванные возникновением общей неисправности, сбоя и т.п., о чём свидетельствует несколько зарегистрированных от пользователей запросов с одинаковой корневой причиной. С массовым запросом можно связать несколько подчинённых запросов.Изменение типа массового запроса недопустимо, если у него есть подчинённые запросы.**Подчинённый запрос переходит в статус, в котором находится массовый запрос. При изменении статуса массового запроса происходит автоматическое изменение статуса всех подчинённых запросов, связанных с массовым. Это наследование осуществляется до тех пор, пока массовый запрос не перейдёт в статус «Разрешён».
В подчинённом запросе отображаются атрибуты, наследуемые из массового запроса. Значения атрибутов копируются из массового при связывании заявок и при изменении атрибутов в массовом запросе до тех пор, пока связь не будет разорвана.Все действия с массовыми запросами выполняются в форме работы с массовыми инцидентами. Форма открывается в карточке запроса при нажатии на иконку «Массовый инцидент». В этой форме можно сделать запрос массовым, связать подчинённые запросы или разорвать с ними связь.![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_19_1354x600.png)
### Связь запросов с массовым запросомС массовым запросом можно связать запросы, выбрав их в форме «Массовый инцидент» в блоке «Добавить связь с подчинёнными инцидентами».При выборе запросов можно воспользоваться поиском или фильтром по услуге. Для выбора одного запроса/запросов установите флажок напротив нужного. После завершения настройки нажмите кнопку «Сохранить».![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_20.png)
#### Разрыв связи с массовым запросомДля разрыва связи с массовым запросом в карточке подчинённого запроса, в блоке «Массовые инциденты» нажмите «Разорвать связь с массовым инцидентом».![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_21.png)
## Создание задачи (проблемы, запроса на изменение) по запросуЧтобы добавить задачу с отдельной работой в рамках запроса, в карточке запроса в блоке «Связанные задачи» нажмите ссылку «Добавить».![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_22.png)Заполните поля в открывшейся форме добавления задачи и нажмите кнопку «Сохранить».![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_23.png)Связь с запросом на изменения и проблемами добавляются аналогичным образом. Подробнее см. в соответствующих разделах.
### Связь с Запросом на изменениеУ запроса на изменение могут быть два типа связей:-Запросы-причины — это инциденты или запросы, из-за которых нужно внести изменения (например, проблемы в оборудовании или структуре). Связь с ними указывается при создании запроса на изменение или в его карточке-Запросы-следствия — это инциденты, которые появились после внедрения изменения и вызваны им. Связь с ними оформляется в карточке инцидента, на вкладке «Запросы на изменение», в разделе «Причины»Для инцидента запрос на изменение — это причина, для самого запроса — инцидент является следствием.
## Переписка по запросуСпециалисты ИТ-службы и пользователи-контрагенты запросов могут обмениваться комментариями по вопросам обслуживания запроса в его карточке, в блоке «Комментарии». Специалисты также может добавлять комментарии только для своих коллег, сделав их приватным. При добавлении комментария, уточняющего у заявителя подробности запроса, необходимо перевести запрос в статус «Ожидает ответа заявителя». В этом случае отсчёт времени на решение запроса приостанавливается до получения ответа на комментарий. При получении ответа заявителя запрос возвращается в работу автоматически. При этом если комментарий добавил ответственный или член его команды, то заявка не возвращается обратно в работу.Чтобы добавить комментарий, в карточке запроса в блоке «Комментарии» начните вводить комментарий. Для перехода в полную форму добавления комментария нажмите «Развернуть».![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_24.png)Поля в форме добавления комментария:-Текст — отредактируйте содержание комментария из шаблона или укажите своё содержание комментария.В текст можно вставить упоминание:Другого сотрудника — после ввода символа «@» выберите сотрудника. Всем выбранным сотрудникам будет отправлено оповещение с текстом комментария. По имени сотрудника из комментария можно будет перейти в карточку сотрудника-Статью Базы знаний — после ввода строки «БЗ» выберите статью Базы знаний. По названию статьи из комментария можно будет перейти в карточку статьи-Приватный: флажок снят — комментарий является общим для пользователей и для специалистов; флажок установлен — комментарий доступен только специалистам-Дополнительные файлы** — при необходимости выберите файл с ПК-Уведомить внешний email — выберите получателя вне компании, которому будет отправлено уведомление о комментарии (в поле необходимо указать почту получателя, если их несколько — отделять адреса запятой)В расширенной форме добавления комментария можно настраивать форматирование текста.![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_25.png)Пользователь получает оповещения об изменения объекта, на который он подписан или в котором он ответственный/контрагент. Он может отвечать на поступающие оповещения. Если пользователь-контрагент в теме ответного сообщения укажет номер запроса, текст сообщения в виде комментария будет добавлен в систему.

## Приостановка и возобновление работ по запросу на обслуживание
### Приостановка работ по запросу на обслуживаниеРаботы по запросу на обслуживание могут быть отложены специалистом на некоторое время.Чтобы отложить запрос на обслуживание, в карточке запроса нажмите кнопку «Отложить».![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_30.png)В форме изменения статуса укажите дату и время, до наступления которых будут отложены работы по запросу, а также причину приостановки работ — в комментарии.![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_31.png)При наступлении даты и времени, указанных в поле «Отложить до», запрос на обслуживание будет автоматически возвращён в работу в статусе «В работе».
### Возобновление работ по запросу на обслуживаниеЗапрос на обслуживание можно вернуть в работу до наступления даты и времени, указанных в поле «Отложить до», для этого в карточке запроса на обслуживание нажмите кнопку «В работу».![](Resources/Images/Инструкция_Администратора_NSD_пирамидка_сайт/Обработка запросов_32.png)
## Закрытие запросаПосле завершения работы специалист переводит запрос в статус «Выполнена». После этого автор запроса получает оповещение о её решении.Статус «Выполнена» означает, что работы по запросу завершены и от пользователя ожидается подтверждение предоставления услуги по запросу. Если контрагент подтверждает, что запрос выполнен, он может поставить оценку выполнению работ.Если в течение N дней от пользователя не поступает никакой информации, то запрос автоматически переводится в статус «Закрыт» («N» настраивается Администратором системы или Технологом).Если контрагент подтверждает, что запрос выполнен, он может оценить выполнение работ, поставив оценку.Статус «Закрыт» является конечным статусом запроса и означает, что выполнение запроса подтверждено.Специалист может возобновить работы по запросу по просьбе автора запроса (кроме заявок в статусе «Закрыт»), также автор сам может возобновить работы над запросом.
